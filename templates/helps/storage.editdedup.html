Edit Deduplication Storage

Here you can edit disk based deduplication backup storage. It is an extended file based storage available in Bacula Enterprise and Inteos Backup.

Storage name - name of your backup storage you edit.

Description - any description of your backup storage.

Dedup index directory - location for the deduplication index database. It should be located on fast storage (i.e. SSD) for best performance.

Dedup engine directory - location for the deduplication data containers and Volumes archive. All backups data will be stored in this location.

Departments - departments name for the storage; you can select multiple departments.

If you change any of Dedup index directory or Dedup engine directory locations you have to move all data from old location to new location unless you will be unable to make any new backup or restore. It requires a manual Storage Daemon restart for fully use of this storage. If doubt please consult it with Inteos support unless you know what you are doing.
