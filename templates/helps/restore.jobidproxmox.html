Restore Proxmox VM data

Here you can select Proxmox Guest VMs to restore. All QEMU type guests are marked with icon and all LXC type quests are marked with icon.

You can select any guest vm name to restore then all VMs with this name will be selected to restore.

You can check what VM ids were stored at a selected vm name by expanding the vm name itself, then you can select a single vmid for restore.

JobIds involved - the list of job ids which will be involved in the restore job.

Restore to directory - when selected the guest vm data will be restored to Proxmox host local directory instead of Proxmox appliance. The default is to restore to Proxmox appliance.

Restore host - the Proxmox host selected for restore (default is the original host).

Proxmox storage - when restore to Proxmox appliance it will save restored guest in the selected Proxmox storage (default is to put guest to its original storage).

Proxmox resource pool - you can assign guest vm into a selected Proxmox resource pool on restore (default is no pool).

Where - when Restore to directory is selected then guest vm data will be saved in the selected directory.

Replace mode - when Restore to directory is selected then it manages replace mode for the restore job.
