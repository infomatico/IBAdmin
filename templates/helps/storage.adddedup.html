Add New Deduplication Storage

Here you can define a new disk based deduplication backup storage. It is an extended file based storage available in Bacula Enterprise and Inteos Backup.

Storage name - name of your new backup storage. It has to be unique in your system.

Description - any description of your backup storage.

Existing Storage - select already defined storage - Bacula Storage Daemon.

Dedup index directory - location for the deduplication index database. It should be located on fast storage (i.e. SSD) for best performance.

Dedup engine directory - location for the deduplication data containers and Volumes archive. All backups data will be stored in this location.

Both Dedup index directory and Dedup engine directory locations must exist on local storage and has to be writable by bacula user. You can't make successful backup if Storage will be unable to write to this location.

When you add a new storage you need to restart Storage Daemon manually for fully use of this storage.
