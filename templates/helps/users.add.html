New User

Here you can define a new user which can have access to IBAdmin.

  • User name - is the name which will be used to login to service.
  • First name/Last name - the name of the user.
  • Email - user email.
  • Password - user password used to authenticate.
  • User type - the basic user level access. There are three acceslevels possible:
    • Superuser - has all permissions and access to all resources by default. You cannot assign any Department or Role to Superuser.
    • Administrator - has all permissions to all resources in his Departments if he is a member of any Department. Has no access to system configuration and statistics.
    • Standard - has no permissions until assigned by Roles. Has access to resources available in his Departments if he is a member of any Department.
  • Departments - select multiple Departments which will be assigned to user.
  • Roles - select multiple permission Roles granted to user.

Info - any user with "Can add users" permission can define new user with access level no higher then himself. When Administrator or Standard user is the member of any Department he has to define departments for new user from his membership. You cannot assign Roles to Administrators.
