New KVM Host Client

Here you can define a new standalone KVM host client. It will be defined as a standard single Bacula File Daemon. To define a new KVM backup host client you have to input its name which should be unique in the system (you will get an error if it is not), description (optional), address which could be an IP or DNS name (if application cannot resolve DNS name you cannot add this client), its department and optionally check for creating a default backup Job for this client.

Client name
name of your new backup client. It has to be unique in your system.
Description
any description of your backup client.
Address
IP address or FQDN of your backup client. It has to be a valid (resolved) address.
Department
department name for the client.
Create default backup Job
when checked a default Job for the Client will be created together with a Client.

Default Job will have the name <client_name>-AllVMGuests and will backup all vmguests available in this Proxmox host.
