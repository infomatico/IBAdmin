New Cluster Node Backup Client

Here you can define a new cluster node backup client. Cluster node backup client is a Bacula File Daemon which shares some properities across all nodes in the cluster.

Client name - name of your new backup client. It has to be unique in your system.

Description - any description of your backup client.

New Cluster name - name of the new cluster component for a node client. It has to be unique in your system.

Existing Cluster - name of the existing cluster component for a node client.

Address - IP address or FQDN of your backup client. It has to be a valid (resolved) address.

OS - an OS type of the defined client.

Create default backup Job - when checked a defaut Job for the Client will be created together with a Client.

Default Job will have the name <client_name>-DefaultJob.
