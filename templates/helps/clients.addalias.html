New Backup Client Alias

Here you can define a new backup client Alias. Backup client Alias is like a symbolic link or CNAME which points to standard Backup Client. To define a new alias backup client you have to input its name which should be unique in the system (you will get an error if it is not), description (optional), select an existing backup client (which could be a standalone or cluster node client) from a list and optionally check for creating a default backup Job for this client.

Alias name - name of your new backup client - Alias. It has to be unique in your system.

Description - any description of your backup client.

Alias to - name of the existing Backup Client (standalone or cluster node) to point to.

Create default backup Job - when checked a defaut Job for the Client will be created together with a Client.

Default Job will have the name <client_name>-DefaultJob.
